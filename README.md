# 🍑 과일 최저가 시즌 알리미

한국농수산식품유통공사(aT)의 "연월별 도,소매가격정보" 공공데이터를 이용해, 과일별로 언제
가장 저렴한지 보여주고 AI에게 가격에 대해 물어볼 수도 있는 Streamlit 앱입니다.

## 주요 기능

### 1. 메인 페이지 (`main.py`)

- **과일 선택 그리드**: 사과·배·복숭아·포도·감귤·단감·바나나·참다래·파인애플·오렌지·자몽·레몬·
  체리·망고·블루베리·아보카도·수박·토마토·딸기·참외 총 20종 (한 줄에 5개씩, 모바일에서는
  3개씩 자동으로 줄바꿈).
- **해시태그 퀵 필터**: "#지금_가장_싼_과일" 등 클릭 한 번으로 관련 과일을 바로 선택.
- **가격 상태 카드**: 신호등(초록/노랑/빨강/회색)으로 지금이 사기 좋은 시점인지 한눈에 표시하고,
  Upstage Solar가 생성한 2줄 분석(실패 시 규칙 기반 문구로 자동 대체), 주요 품종·대표 산지 정보를
  함께 보여줍니다. 출하 시기가 아니라 데이터가 없을 때는 "지금은 출하 시기가 아니에요"로
  정확하게 안내합니다.
- **최근 12개월 가격 추이 그래프**: 얇은 선 + 속이 빈 원형 마커의 미니멀한 디자인. 최저가 달은
  은은한 배경 면 + 점선 테두리로, 지금 달은 점선 세로선 + 점 하나로 표시해서 두 시점이 겹쳐도
  서로 지워지지 않습니다. y축 숫자는 감추고 정확한 가격은 hover 툴팁으로 확인합니다.
- **소매/도매 가격 정규화**: API의 `unit`·`unit_sz` 필드를 읽어 가능하면 **1kg당 가격**으로
  통일해서 비교합니다(개·상자처럼 무게 단위가 아닌 경우는 억지로 환산하지 않고 원본 단위를
  그대로 두고 화면에 "환산 불가"로 표시). "🔍 원본 데이터로 확인" 표에서 원본 가격·단위·환산
  기준을 직접 검증할 수 있습니다.
- **월별 가격표**: 접어두는 표로, 값은 가운데 정렬, 최저가·지금 달에는 강조 표시.
- **스마트 구매팁**: 품종별 구매팁 / 좋은 과일 고르는 법 / 구매 채널 안내 3장의 카드. Solar로
  과일별 맞춤 문구를 생성하고, 실패 시 20개 과일 각각의 규칙 기반 문구로 자동 대체됩니다.
- Solar를 이용해 페이지 제목·부제목·해시태그·출처 문구도 매일 한 번 다듬어서 표시합니다
  (실패 시 기본 문구 사용).

### 2. AI 챗봇 페이지 (`pages/1_AI_챗봇.py`)

- 왼쪽에는 미리 정해둔 질문 10개("왜 겨울 수박은 비쌀까?" 등) 버튼, 오른쪽에는 채팅창.
- Upstage Solar(`solar-open2`)로 실시간 스트리밍 답변, 대화 기록은 세션에 저장되어 이어집니다.
- "너는 따뜻하고 친절한 데이터 분석 선생님이야. 반드시 순수 한국어로만 답해"라는 시스템
  프롬프트를 사용합니다.
- API 호출이 실패해도 에러 화면 대신 친절한 한국어 안내 메시지가 말풍선으로 표시됩니다.

## 데이터 소스

- 서비스: 한국농수산식품유통공사_연월별 도,소매가격정보 (`B552845/perYearMonth/price`)
- 필수 파라미터: `serviceKey`, `returnType`, `pageNo`, `numOfRows`,
  `cond[exmn_ym::GTE]`, `cond[exmn_ym::LTE]`
- 선택 파라미터: `cond[ctgry_cd::EQ]`(부류코드), `cond[item_cd::EQ]`(품목코드)
- 정확한 필드명·부류/품목 코드는 함께 제공되는 `api_des.xlsx`, `price_code.xlsx`를 그대로
  참조했습니다. **이 두 파일은 배포 시 `main.py`와 같은 폴더에 있어야 품목 코드 조회가
  정상 동작합니다.**
- API 호출이 실패하거나(키 미등록, 네트워크 오류 등) 응답이 비어 있으면, 계절성을 반영한
  데모 데이터로 자동 대체되어 화면이 비지 않습니다(이 경우 "데모 데이터" 안내가 표시됩니다).

## 기술 스택

- **Streamlit** — UI 프레임워크 (멀티페이지: `main.py` + `pages/`)
- **Plotly** — 가격 추이 그래프
- **pandas** — 데이터 가공
- **openai (Python SDK)** — Upstage Solar(`solar-open2`) 호출용 (Solar는 OpenAI 호환 API를
  제공하므로 `base_url="https://api.upstage.ai/v1"`로 openai 라이브러리를 그대로 사용)
- **requests** — 공공데이터포털 API 호출
- **openpyxl** — `price_code.xlsx` / `api_des.xlsx` 읽기

## 파일 구조

```
.
├── main.py                       # 메인 페이지 (가격 조회 · 그래프 · 구매팁)
├── pages/
│   ├── 00_과일 인사이트.py        # 가격 조회 · 그래프 · 구매팁 서브페이지
│   └── 01_AI 가격 예측.py         # AI 챗봇 서브페이지
├── price_code.xlsx                # 부류/품목/품종/등급/시군구 코드표 (원본 파일명 유지 필요)
├── api_des.xlsx                   # API 요청/응답 필드 명세 (원본 파일명 유지 필요)
├── requirements.txt
└── secrets.toml.example           # secrets.toml 작성용 템플릿
```

## 실행 방법

### 1. 로컬 실행

```bash
pip install -r requirements.txt
mkdir -p .streamlit
cp secrets.toml.example .streamlit/secrets.toml   # 값을 실제 키로 채워넣기
streamlit run main.py
```

### 2. Streamlit Cloud 배포

1. 이 폴더(`main.py`, `pages/`, `price_code.xlsx`, `api_des.xlsx`, `requirements.txt`)를 그대로
   깃허브 저장소에 올립니다. **엑셀 파일 두 개의 원본 파일명을 바꾸면 안 됩니다.**
2. Streamlit Cloud에서 새 앱을 만들 때 메인 파일로 `main.py`를 지정합니다.
3. App settings → Secrets에 아래 내용을 등록합니다 (`secrets.toml.example` 참고).

```toml
FRUITS_API_KEY = "공공데이터포털에서 발급받은 인증키(디코딩키)"
SOLAR_API_KEY = "Upstage Console에서 발급받은 Solar API 키"
```

- `FRUITS_API_KEY`가 없어도 앱은 죽지 않고 데모 데이터로 동작합니다.
- `SOLAR_API_KEY`가 없어도 가격 분석·구매팁·페이지 문구는 규칙 기반 문구로, AI 챗봇은
  친절한 한국어 안내 메시지로 자동 대체됩니다.

## 알아두면 좋은 점 (한계 및 참고사항)

- **소매/도매 가격 비교**: API의 `unit`/`unit_sz`가 무게 단위(kg·g)일 때만 1kg당 가격으로
  정확히 환산됩니다. 개·상자 단위는 무게로 환산할 근거가 없어 원본 단위를 그대로 쓰며, 그래도
  가격이 역전되어 보인다면 실제 시세 차이일 수 있습니다 — "🔍 원본 데이터로 확인" 표에서
  직접 검증해 보세요.
- **대표 산지 정보**는 공공데이터 API 응답에 없는 항목이라, 일반적으로 널리 알려진 주산지를
  기준으로 별도 정리한 참고용 정보입니다.
- **과일 이모지**: 유니코드에 전용 이모지가 없는 과일(자몽·단감·참외 등)은 형태·색이 가장
  비슷한 이모지로 대체했습니다.
- Solar 호출(가격 분석 1회/시간, 페이지 문구 1회/일, 구매팁 1회/일)은 `st.cache_data`로
  캐싱되어 있어 불필요하게 자주 호출되지 않습니다.
